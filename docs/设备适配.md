# 设备适配

> 本文档列出所有支持的设备、平台映射、源码 submodule 映射与配置详情。

## 设备总览

共 17 个编译配置,覆盖高通 IPQ、联发科 Filogic、Airoha、ARMv8、x86 五大平台。

## 源码 submodule 映射

每个设备的 `compilecfg/<device>.ini` 中 `BUILD_DIR` 字段即 **source-id**,对应 `sources/<source-id>` submodule。本项目纳入 6 个上游源:

| source-id | 上游仓库 | 分支 | 用途 | .gitmodules 注册名 |
|-----------|----------|------|------|-------------------|
| `immortalwrt` | [immortalwrt/immortalwrt](https://github.com/immortalwrt/immortalwrt) | master | 官方 ImmortalWRT(通用平台) | sources/immortalwrt |
| `imm-nss` | [VIKINGYFY/immortalwrt](https://github.com/VIKINGYFY/immortalwrt) | main | IPQ60xx/IPQ807x NSS 加速优化 | sources/imm-nss |
| `libwrt` | [LiBwrt/openwrt-6.x](https://github.com/LiBwrt/openwrt-6.x) | main-nss | LibWrt 分支(内核 6.x,NSS) | sources/libwrt |
| `libwrt-k612` | [LiBwrt/openwrt-6.x](https://github.com/LiBwrt/openwrt-6.x) | 25.12-nss | LibWrt 25.12 NSS 源 | sources/libwrt-k612 |
| `imm-mt798x` | [padavanonly/immortalwrt-mt798x](https://github.com/padavanonly/immortalwrt-mt798x) | openwrt-21.02 | MT798x 专属(21.02 基础) | sources/imm-mt798x |
| `airoha-wrt` | [ZqinKing/immortalwrt](https://github.com/ZqinKing/immortalwrt) | w1701k | Airoha W1701K 定制分支 | sources/airoha-wrt |

**注意**:`airoha-wrt` 的上游是 ZqinKing 账号下的 immortalwrt fork,分支 `w1701k` 是 W1701K 专属定制分支,属功能性依赖。

## 平台专属定制

### IPQ 平台(IPQ60xx / IPQ807x)

由 [system.sh](../wrt_core/modules/system.sh) 中的多个函数处理:

- `remove_something_nss_kmod`:移除不必要的 NSS kmod 依赖
- `update_affinity_script`:安装 IRQ 亲和性脚本
- `update_ath11k_fw`:同步 ath11k 固件 Makefile 为 ddwrt 版
- `change_cpuusage`:使用 [patches/cpuusage](../wrt_core/patches/cpuusage) 显示 CPU + PPE 计数
- `update_nss_diag`:替换 nss_diag.sh
- `update_nss_pbuf_performance`:NSS pbuf 性能调优
- `update_script_priority`:调整 NSS init 脚本启动顺序

配置层面,IPQ 设备会额外追加 [nss.config](../wrt_core/deconfig/nss.config)。

### MediaTek Filogic 平台(MT7986)

- `change_cpuusage`:使用 [patches/hnatusage](../wrt_core/patches/hnatusage) 显示 CPU + HNAT 计数

### x86 平台

- distfeeds 架构修正(`aarch64_cortex-a53` → `x86_64`)
- 产物包含 EFI 镜像

## 新增设备指南

### 1. 确定上游 source-id

设备属于哪个平台?选择对应的 source-id:
- IPQ60xx/IPQ807x → `imm-nss`(ImmWrt 版)或 `libwrt`(LibWrt 版)
- MT7986 → `immortalwrt` 或 `imm-mt798x`
- 其他 → `immortalwrt`

若需要全新的上游源(不在上表 6 个中):
1. 在 `.gitmodules` 添加 submodule 声明:
   ```
   [submodule "sources/<new-source-id>"]
       path = sources/<new-source-id>
       url = https://github.com/<upstream>.git
       branch = <branch>
   ```
2. 在 [metadata/external-dependencies.tsv](../metadata/external-dependencies.tsv) 添加一行 `upstream-source` 记录
3. 执行 `git submodule add ...` 初始化

### 2. 创建 ini 配置

`wrt_core/compilecfg/<device>.ini`:

```ini
REPO_URL=https://github.com/<upstream>.git
REPO_BRANCH=<branch>
BUILD_DIR=<source-id>
```

`REPO_URL`/`REPO_BRANCH` 必须与 `.gitmodules` 中对应 submodule 一致(用于 `WRT_USE_DYNAMIC_CLONE` 回退模式与 source-lock 记录)。

### 3. 创建 config 配置

`wrt_core/deconfig/<device>.config`,至少包含:

```
CONFIG_TARGET_<subsystem>=y
CONFIG_TARGET_<subsystem>_<soc>=y
CONFIG_TARGET_MULTI_PROFILE=y
CONFIG_TARGET_PER_DEVICE_ROOTFS=y
CONFIG_TARGET_DEVICE_<subsystem>_<soc>_<DEVICE_ID>=y
CONFIG_TARGET_DEVICE_PACKAGES_<subsystem>_<soc>_<DEVICE_ID>="<per-device packages>"
```

参考现有设备 config(如 [jdcloud_ipq60xx_immwrt.config](../wrt_core/deconfig/jdcloud_ipq60xx_immwrt.config))。

### 4. CI 注册

在 [.github/workflows/build_wrt.yml](../.github/workflows/build_wrt.yml) 的 `inputs.model.options` 列表添加新设备名。

### 5. 初始化 submodule 并测试

```bash
git submodule update --init --depth 1 -- sources/<source-id>
./build.sh <device> debug    # 验证 defconfig
./build.sh <device>          # 完整编译
```

## ImmWrt vs LibWrt 的区别

部分设备同时提供 ImmWrt 与 LibWrt 两个版本:

| 维度 | ImmWrt 版 | LibWrt 版 |
|------|-----------|-----------|
| source-id | imm-nss / immortalwrt | libwrt / libwrt-k612 |
| 上游 | VIKINGYFY/immortalwrt 或 immortalwrt | LiBwrt/openwrt-6.x |
| 内核 | 跟随上游 | 6.x(通常更新) |
| 兼容性 | 与 ImmortalWRT 生态一致 | 独立分支,部分包可能不兼容 |
| NSS | 支持 | 支持 |

选择建议:
- 追求稳定 → ImmWrt 版
- 需要新内核特性 → LibWrt 版
- 不确定 → ImmWrt 版
